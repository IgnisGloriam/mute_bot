# Mute Bot

Telegram-бот, автоматически выдающий мут (запрет на отправку сообщений) новым участникам группы.  
Разработан на базе библиотеки [aiogram 3.x](https://docs.aiogram.dev/).

## Возможности

- Отслеживание событий вступления пользователя в группу (`chat_member`).
- Автоматическое ограничение прав нового участника (`can_send_messages=False`).
- Команда `/check` – отвечает "Check" в личных сообщениях (для проверки работоспособности бота).

## Технологии

- Python 3.9+
- aiogram 3.x
- pydantic-settings (загрузка конфигурации из `.env`)

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/IgnisGloriam/mute_bot.git
cd mute-bot
```

### 2. Создание и активация виртуального окружения
```bash
python -m venv venv
source venv/bin/activate      # для Linux/macOS
venv\Scripts\activate         # для Windows
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 4. Настройка переменных окружения
Создайте в корне проекта файл `.env` и укажите в нём токен вашего бота:
```
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
```
Получить токен можно у [BotFather](https://t.me/botfather).

### 5. Запуск бота
```bash
python mute_bot.py
```

## Структура проекта

- `mute_bot.py` – основной код бота (обработчики событий, запуск поллинга).
- `config_reader.py` – модуль загрузки конфигурации через Pydantic Settings.
- `.env` – файл с переменными окружения (не хранится в репозитории).
- `requirements.txt` – список зависимостей.

## Использование

1. Добавьте бота в группу и назначьте его **администратором** (это необходимо для применения ограничений).
2. При вступлении нового участника бот автоматически запретит ему отправлять сообщения.
3. Напишите боту в личные сообщения команду `/check` – он ответит `Check`.

## Важные замечания

- **Права администратора**: чтобы бот мог менять права участников, он должен быть администратором группы с включённой возможностью «блокировать пользователей».

- Команда `/check` обрабатывается только в личных чатах с ботом.
